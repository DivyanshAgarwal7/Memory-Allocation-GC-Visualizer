The GC lifecycle

Every run follows the same three steps.

This is the actual loop the simulator runs — the same one your operating system runs, just slowed down and made visible.

01

Allocate

Request a number of bytes. The active strategy picks a free block and splits off any leftover space.

02

Mark & Sweep

Mark which objects are still reachable. Sweep frees every used block that wasn't marked.

03

Coalesce

Adjacent free blocks merge automatically, undoing fragmentation before the next allocation.
